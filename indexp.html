About

I started on an API development team, but testing is where I found the work I actually want to keep doing — finding the gap between what an endpoint should return and what it actually returns, and making sure it doesn't ship that way.

At TCS, I worked inside an Agile team testing APIs for an enterprise financial application built for MasterCard. My day-to-day: pick up assigned endpoints, write and run test cases against them in Postman and Insomnia, check the response against the database in PostgreSQL, and log anything that didn't match. When something broke, I tracked it through to a verified fix with the dev team.

I've also sat in on client calls — not to gather requirements, that's handled by seniors — but to confirm with the client that a specific API was behaving the way it should, and to document that outcome clearly.

Outside work, I've been self-teaching the cloud and CI/CD side of QA — AWS, Azure, and Jenkins fundamentals — because I want to grow toward SDET and cloud-aware testing roles, not stay narrowly manual.

A note on scope: my hands-on testing experience is specifically in API testing — not manual functional/regression testing. I'm upfront about that distinction because I'd rather be accurate than impressive.
